# kaitoy's Tech Blog

## プロジェクト概要
kaitoyの技術ブログサイトのプロジェクト。Hugoでビルドして、GitHub Pagesで公開する。

## ディレクトリ構造
このプロジェクトのディレクトリ構造は以下。

- content: ブログコンテンツのソース
    - about: aboutページのコンテンツのソース
    - post: 各記事のコンテンツのソース
- themes: ブログのテーマ
- layouts: テーマからカスタマイズしたレイアウト
    - shortcodes: HugoのShortcode
- static: 画像などの静的なアセット
- docs: ビルド成果物
- resources: ビルド成果物

## ブログ記事執筆ワークフロー

1. **企画・構想の把握**
    - `content/post/{タイトル}.md` の草案を読み、主題、キーワード、記事のゴールを把握する。
2. **ガイドラインの確認**
    - `README.md` の `AI-Writer's Guide` を読み、スタイルや規約を再確認する。
3. **主題に関する情報収集と理解**
    - 主題や主要なキーワードについて、正確性を期すためにインターネットで情報を収集・調査する。
    - **[重要]** もし主題が不明確、または複数の解釈が可能な場合は、ユーザーに質問して意図を明確にする。
4. **構成案の作成と提案**
    - 収集した情報と草案の意図に基づき、記事の詳細な構成案（タイトル、各セクションの見出し、概要）を作成する。
    - ユーザーに構成案を提示し、承認を得る。これにより、執筆後の大幅な手戻りを防ぐ。
5. **記事の執筆**
    - 承認された構成案とガイドラインに従って、記事を執筆する。
    - 草案の内容を補完、または全面的に書き直す。
    - 記事は日本語で執筆する。
6. **推敲と最終確認**
    - 執筆した記事が、構成案、ガイドライン、そしてユーザーの意図に沿っているかを確認する。
    - 誤字脱字、技術的な誤りがないか最終チェックを行う。
    - 必要に応じて記事を修正する。
